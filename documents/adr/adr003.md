# ADR 003: Implementação de Cache Distribuído (Redis)

## Contexto
Funcionalidades como listagem de filmes e visualização de assentos geram carga excessiva no banco de dados relacional (PostgreSQL).

Riscos identificados:
* **Gargalo:** O banco de dados pode travar em dias de estreias de grandes filmes.
* **Latência:** O tempo de resposta para carregar o mapa de assentos está lento.

## 2. Decisão
Implementação de uma camada de **Cache Distribuído** utilizando **Redis**, integrado via `@nestjs/cache-manager`.

### Detalhes da Implementação:
* **Escopo:** Microsserviços de *Catalog* e *Session*.
* **Estratégia:** A aplicação busca no Redis; se der *miss*, busca no banco e popula o cache.

## 3. Alternativas Rejeitadas
**Cache em Memória (In-Memory / RAM Local)**
* **Motivo da Rejeição:** Se tivermos múltiplas réplicas do mesmo serviço, o cache local causaria inconsistência de dados entre os containers.

## 4. Consequências

### Benefícios
* Redução na carga de leitura direta no PostgreSQL.
* Tempo de resposta da API reduzido.
* Maior resiliência do sistema durante picos de tráfego.
